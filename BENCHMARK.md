## Key Insights

1. **Token savings are the dominant factor.** Every cached file read costs ~0 input tokens — the difference between paying for 100 MB of file content and paying for nothing.
2. **Time savings grow with command cost.** Cheap commands (`echo`, `pwd`) save less absolute time than expensive ones (`find`, `du`). Slow commands (>50ms raw) see the biggest speedup.
3. **Immediate warmup.** One initial cache miss per file populates the cache. Every subsequent read is a hit — no gradual learning curve.
4. **File size cross-over point.** Files > 10 MB see both time AND token savings. Below 10 MB, the token saving alone justifies the cache.
5. **TTL tuning matters.** Short TTL (30s) for mutable commands like `git status`; long TTL (3600s) for stable commands like `hostname`. The defaults work for most cases.
